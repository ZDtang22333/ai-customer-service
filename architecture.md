## 二、RAG 全流程详解

```
1. 文档加载 (Loader)
   ┌─────────────────────────────────────┐
   │ DirectoryLoader("knowledge/")       │
   │ 输入: 目录路径 (str)                │
   │ 输出: List[Document]                │
   │       Document.page_content (str)   │
   │       Document.metadata (dict)      │
   └─────────────────────────────────────┘
                     │
                     ▼
2. 文本切分 (Splitter)
   ┌─────────────────────────────────────┐
   │ RecursiveCharacterTextSplitter      │
   │ chunk_size=500, overlap=50          │
   │                                     │
   │ 为什么500字？                        │
   │ - 太短：语义不完整，检索不准确       │
   │ - 太长：包含太多无关信息，稀释相关性 │
   │ - 500字 ≈ 一个段落，语义完整         │
   │                                     │
   │ 为什么50字重叠？                     │
   │ - 防止一句话被切断在两个块里         │
   │ - 保持上下文连贯                     │
   └─────────────────────────────────────┘
   输入: List[Document]
   输出: List[Document] (chunks)
                     │
                     ▼
3. 向量化 (Embedding)
   ┌─────────────────────────────────────┐
   │ HuggingFaceEmbeddings               │
   │ 模型: BAAI/bge-small-zh-v1.5        │
   │                                     │
   │ "笔记本电池能用多久"                 │
   │     → [0.012, 0.096, -0.023, ...]   │
   │     → 512 维向量                    │
   │                                     │
   │ 语义相似 → 向量距离近               │
   │ 语义不同 → 向量距离远               │
   └─────────────────────────────────────┘
   输入: str (文本)
   输出: List[float] (512维向量)
                     │
                     ▼
4. 存储 (ChromaDB)
   ┌─────────────────────────────────────┐
   │ Chroma.from_documents()             │
   │                                     │
   │ 存储内容:                           │
   │ - 文本 (page_content)               │
   │ - 向量 (embedding)                  │
   │ - 元数据 (metadata)                 │
   │                                     │
   │ 持久化到: ./chroma_db               │
   └─────────────────────────────────────┘
   输入: List[Document], Embeddings
   输出: Chroma (向量数据库)
                     │
                     ▼
5. 检索 (Retrieval)
   ┌─────────────────────────────────────┐
   │ 混合检索 = BM25 + 语义搜索          │
   │                                     │
   │ BM25: 关键词匹配                    │
   │   "退货要几天" → 找包含"退货""几天" │
   │   原理: 词频(TF) × 逆文档频率(IDF)  │
   │                                     │
   │ 语义搜索: 意思相近                   │
   │   "不想要了" → 找到"退货政策"       │
   │   原理: 向量余弦相似度              │
   │                                     │
   │ 合并去重 → List[Document]           │
   └─────────────────────────────────────┘
   输入: query (str)
   输出: List[Document]
```

```
┌─────────────────────────────────────────────────┐
│              RAGAS 评估流程                      │
│                                                 │

│  ┌─────────────────────────────────────────┐    │

│  │ Context Recall: 是否找全了相关文档       │    │
│  └─────────────────────────────────────────┘    │
│                     │                           │
│                     ▼                           │
│  4. 输出评估报告                                │
│  ┌─────────────────────────────────────────┐    │
│  │ Faithfulness:      0.89                  │    │
│  │ Answer Relevance:  0.92                  │    │
│  │ Context Precision: 0.85                  │    │
│  │ Context Recall:    0.88                  │    │
│  └─────────────────────────────────────────┘    │
└─────────────────────────────────────────────────┘
```

### 评估指标说明

| 指标 | 衡量什么 | 目标值 | 不合格优化方向 |
|------|----------|--------|----------------|
| Faithfulness | 回答是否忠于文档 | > 0.85 | 优化 Prompt |
| Answer Relevance | 回答是否和问题相关 | > 0.80 | 优化 Prompt |
| Context Precision | 检索的文档是否相关 | > 0.80 | 提高阈值 |
| Context Recall | 是否找全了相关文档 | > 0.85 | 增加候选数 |

## 七、面试话术模板

### 项目介绍（30秒）
> 我做了一个智能电商客服系统，基于 LangChain + RAG + Agent。
> 核心功能两个：一是用 RAG 做知识库问答，用户问产品问题能从文档里检索回答；
> 二是用 Agent 做业务操作，比如查订单、退款，能调用工具。

### RAG 流程（1分钟）
> RAG 分四步：加载、切分、向量化、检索。
> 加载用 DirectoryLoader 读取 txt 文档；
> 切分用 RecursiveCharacterTextSplitter，500字一块，50字重叠，保证语义完整；
> 向量化用 bge-small-zh 本地模型，转成 512 维向量存入 ChromaDB；
> 检索时用混合检索，BM25 找关键词、语义搜索找意思相近的，合并后重排序取 Top-3。

### 混合检索（30秒）
> 混合检索结合了 BM25 和语义搜索。
> BM25 靠词频匹配，用户说"退货"能精确找到包含这个词的文档；
> 语义搜索靠向量距离，用户说"不想要了"能理解成退货的意思。
> 两者结果合并去重，再用 CrossEncoder 重排序，取最相关的 3 个文档。

### 父子块检索（20秒）
> 父子块检索解决上下文丢失问题。
> 切分时同时生成父块（1000字）和子块（200字）；
> 检索时用子块匹配更精确，返回父块上下文更完整。

### 阈值截断（20秒）
> 阈值截断过滤不相关文档。
> 检索后计算每个文档和问题的相似度，低于阈值的丢弃；
> 这样可以避免不相关文档污染 LLM 输入。

### Agent 机制（30秒）
> Agent 用 ReAct 模式：思考、行动、观察循环。
> 用户说"查订单12345"，Agent 思考需要调用 query_order 工具，
> 执行工具拿到订单信息，观察结果后组织语言回复用户。
> 如果信息不完整，会主动追问。

### RAGAS 评估（20秒）
